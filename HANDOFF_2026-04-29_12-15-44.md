First version implemented a "chunking" approach where students subtract arbitrary multiples of the divisor from the dividend. This was **replaced entirely** after the user provided a reference video and image showing the real Danish ballonmetoden.

### 2. Visual Redesign to Match Danish Ballonmetoden (v2)

- **Balloon (oval)** at the top containing the **divisor**
- **Vertical stem/line** going down from the balloon
- **Left of stem**: working numbers (current number being divided)
- **Right of stem**: chosen multipliers (how many times the divisor fits)
- **Underlines** separating digit groups

### 3. Added "Bringdown" Phase with Visual Explanation (v3)

The transition between steps was confusing — e.g. "27" appeared without explanation. Added an intermediate **bringdown phase** between divide steps that shows:

- A calculation annotation box: `1 × 4 = 4` → `6 − 4 = 2`
- A bring-down explanation: "Tag 7 ned → **27**"
- A "Næste →" button (also responds to Enter key)
- Danish TTS narration of the full calculation

**CSS additions:** `.bd-annotation`, `.bd-annotation-calc`, `.bd-anno-step`, `.bd-anno-arrow`, `.bd-annotation-bring`, `.bd-bringdown-highlight`

### 4. Fixed TTS Pronunciation

Replaced `×` (`\u00d7`) and `−` (`\u2212`) with **"gange"** and **"minus"** in all spoken/readable text so Danish TTS pronounces them correctly. Visual-only HTML (annotation box) still uses math symbols.

### 5. Digit Highlight During Bringdown

During the bringdown phase, the specific digit being brought down from the dividend is now **highlighted in teal** in:
1. The problem title (e.g. "64**4** : 4 = ???")
2. The annotation text (e.g. "Tag **4** ned → 4")
3. The stem row (bringdown highlight, already existed)

**CSS addition:** `.bd-digit-bring` — teal color, bold, subtle background

### 6. Forgiving Algorithm Rewrite (v4 — current version)

**Major algorithm change.** The balloon method now allows students to choose ANY valid multiplier (not just the maximum), making it more forgiving than long division:

- **Any valid pick is correct**: For "how many times does 4 go into 27?", buttons 1–6 are ALL shown and ALL accepted
- **Remainder loop**: If the student picks less than max, the remainder stays >= divisor, so they get another row to keep dividing the same number (no bring-down yet)
- **Bring-down only when remainder < divisor**: Once remainder is small enough, next digit is brought down
- **Quotient accumulator**: Tracks place value correctly (`acc += pick`, then `acc *= 10` on each bring-down), ensuring the final answer is always correct regardless of path
- **No wrong answers**: Every pick triggers `onCorrect()`, feedback is always positive ("Godt valgt!")
- **Dynamic step count**: Number of rows/steps depends on student choices (picking 1 each time = many rows; picking max each time = fewest rows)

**State model:**
```javascript
state = {
  dividend, divisor, digits[],
  nextDigitIdx,       // index of next digit to bring down
  currentNumber,      // number currently being divided
  quotientAcc,        // accumulated quotient (tracks place value)
  rows[],             // { number, multiplier, product, remainder, bringdownAfter }
  currentPhase,       // 'divide' | 'bringdown' | 'done'
  bringdownInfo,      // { remainder, digit, digitIdx, newNumber }
  finalRemainder,
  ...
}
```

**Phase flow:**
1. `divide` → student picks multiplier → if remainder >= divisor: loop to `divide` (no bringdown); if remainder < divisor AND more digits: `bringdown`; if no more digits: `done`
2. `bringdown` → shows annotation, "Næste" button → `divide`
3. `done` → shows final answer, "Nyt regnestykke" / "Tilbage til menu"

## Decisions Made

- **Balloon visual matches reference image**: Oval balloon at top with divisor, vertical stem, two-column layout (left=numbers, right=multipliers)
- **CSS prefix `bd-`** used for all balloon division classes (mirroring `ld-` for long division)
- **`skipTableSelector: true`**: No multiplication table selector needed (like long division)
- **Options exclude 0**: Only 1 through `floor(currentNumber/divisor)` shown as buttons — picking 0 would cause infinite loop
- **Auto-bring-down for small first digits**: If first digit < divisor (e.g. 312÷4, first digit 3 < 4), digits are automatically combined until >= divisor
- **All picks are "correct"**: No `onWrong()` calls since the balloon method accepts any valid multiplier — this is the key pedagogical difference from long division
- **Underlines only between digit groups**: Lines appear after rows where a bringdown happened, NOT between rows within the same digit group (when student picks less than max)
- **TTS uses words not symbols**: "gange" and "minus" instead of × and − for correct Danish pronunciation

## Known Issues/Warnings

- **Problem generation** ensures `firstDigit >= divisor` for easy mode, so the first digit always fits at least once. For hard mode (2-digit divisor), the first 1-2 digits might be < divisor, handled by auto-bring-down.
- **Progress dots** track digits of the dividend (brought-down count), not actual steps. With non-max picks, there are more rows than dots. The dots may be slightly misleading — consider removing or changing to a step counter.
- **Large number of option buttons**: For hard mode with 2-digit divisors, `floor(currentNumber/divisor)` could be large (e.g. 99÷11 = 9 buttons). This is manageable but could be crowded on mobile.
- **Quotient accumulator math**: The accumulator uses `acc += pick` and `acc *= 10` on bringdown. This has been verified logically but should be tested with edge cases (e.g. picking 1 for every step of 644÷4 should yield 161).

## Direction of Development

This is a kid-friendly Danish multiplication tables practice app with now **8 game modes**:
1. **Udfyld Svaret** (Fill in the Blank)
2. **Vælg det Rigtige Svar** (Multiple Choice)
3. **Find den Manglende Faktor** (Missing Factor)
4. **Tidsudfordring** (Speed Challenge)
5. **Hukommelsesspil** (Memory Game)
6. **Hele Tabellen** (Table Sequence)
7. **Lang Division** (Long Division) — strict, digit-by-digit
8. **Ballonmetoden** (Balloon Division) — forgiving, any-valid-multiplier *(NEW)*

Architecture: Single-page vanilla JS app (no frameworks/build tools). Game modes register via `window.GameModes` as separate script files. Shared state in `window.AppState`, shared logic in `window.App`. Danish TTS via Web Speech API (prefers macOS "Sara" voice). Persistence via localStorage. Gold stars for correct answers, orange stars for mistakes (though balloon method has no mistakes by design).
